56 /// @enum Event
57 ///
58 /// @brief Event types are returned by most interactive elements. These are
59 /// flags, because multiple events may occur during one frame, and thus should
60 /// be tested using a Bitwise operators (`&`, `|`, etc.).
61 ///
62 /// For example, it is not uncommon for the value to be
63 /// `kEventWentDown | kEventWentUp` (or `3`), if the click/touch was quicker
64 /// than the current frametime.
65 ///
66 /// You can then check if a specific event occured using the Bitwise AND (`&`)
67 /// operator. For instance, given an `Event my_event`, you could check if
68 /// the `kEventWentDown` Event happened in that frame with
69 /// `if((my_event & kEventWentDown) != 0)`.
70 ///
71 /// **Enumerations**:
72 ///
73 /// * `kEventNone` (`0`) - No Event occured in the frame. (This is also returned
74 /// by all elements during the layout pass.
75 /// * `kEventWentUp` (`1`) - Pointing device (or button) was released this
76 /// frame, while over this element. (This only triggers
77 /// if the element was also the one to receive the
78 /// corresponding `kEventWentDown`.
79 /// * `kEventWentDown` (`2`) - Pointing device is currently being held down on
80 /// top of this element. You're not guaranteed to
81 /// also receive a `kEventWentUp`, as the pointing
82 /// device may have moved to another element (or
83 /// no element) before the frame ends.
84 /// * `kEventIsDown` (`4`) - Pointing device is currently being held down on
85 /// top of this element. You're not guaranteed to
86 /// receive this event between `kEventWentDown` and
87 /// a `kEventWentUp`. That occurs only if the event
88 /// spans multiple frames. This only triggers for
89 /// the element that the corresponding `kEventWentDown`
90 /// fired on.
91 /// * `kEventStartDrag` (`8`) - Pointing device started dragging this frame
92 /// while over this element. The element is expected
93 /// to call `CapturePointer()` to receive the drag
94 /// event continuously, even if the pointer goes off
95 /// of the element.
96 /// * `kEventEndDrag` (`16`) - Pointing device finished dragging in this frame.
97 /// * `kEventIsDragging` (`32`) - Pointing device is currently in dragging mode.
98 /// * `kEventHover` (`64`) - Pointing device is currently over the element, but
99 /// not pressed. This event does NOT occur on touch
100 /// screen devices. It only occurs for devices that use
101 /// a mouse (or a gamepad that emulates a mouse when
102 /// selecting). As such, it is good to show a subtle
103 /// form of highlighting upon this event. However, the
104 /// UI should not rely on it to function.
105 ///
106 /// For example, a typical drag operation would receive the following events
107 /// in this sequence: `kEventWentDown` --> `kEventIsDown` (until the pointer
108 /// motion exceeds a threshold) --> `kEventStartDrag` --> `kEventIsDragging`
109 /// --> `kEventEndDrag`.
110 ///
111 enum Event {
112  kEventNone = 0,
113  kEventWentUp = 1,
114  kEventWentDown = 2,
115  kEventIsDown = 4,
116  kEventStartDrag = 8,
117  kEventEndDrag = 16,
118  kEventIsDragging = 32,
119  kEventHover = 64,
120 };

163 /// @enum Layout
164 ///
165 /// @brief Specify how to layout a group.
166 ///
167 /// Elements can be positioned either horizontally or vertically. The elements
168 /// can be aligned on either side, or centered.
169 ///
170 /// For example, `kLayoutHorizontalTop` indicates that the elements are layed
171 /// out from left-to-right, with items of uneven height being aligned from the
172 /// top.
173 ///
174 /// In this example, we have three elements: `A` with a height of 3, `B` with
175 /// a height of 1, and `C` with a height of 2. So we lay the elements out from
176 /// left to right in the order `A`->`B`->`C`, and we align them along the top.
177 ///
178 /// That layout would look like this:
179 ///
180 /// A @htmlonly &nbsp @endhtmlonly B @htmlonly &nbsp @endhtmlonly C
181 ///
182 /// A @htmlonly &nbsp&nbsp&nbsp&nbsp&nbsp&nbsp @endhtmlonly C
183 ///
184 /// A
185 ///
186 /// **Enumerations**:
187 ///
188 /// * `kLayoutHorizontalTop` (`5`) - Lay out the elements horizontally, aligning
189 /// elements of uneven height along the top.
190 /// * `kLayoutHorizontalCenter` (`6`) - Lay out the elements horizontally,
191 /// aligning elements of uneven height along
192 /// the center.
193 /// * `kLayoutHotizontalBottom` (`7`) - Lay out the elements horizontally,
194 /// aligning elements of uneven height along
195 /// the bottom.
196 /// * `kLayoutVerticalLeft` (`9`) - Lay out the elements vertically, aligning
197 /// elements of uneven width along the left.
198 /// * `kLayoutVerticalCenter` (`10`) - Lay out the elements vertically, aligning
199 /// elements of uneven width along the center.
200 /// * `kLayoutVerticalRight` (`11`) - Lay out the elements vertically, aligning
201 /// the elements of uneven width along the
202 /// right.
203 /// * `kLayoutOverlay` (`14`) - Lay out the elements on top of one another, from
204 /// the center.
205 enum Layout {
206  kLayoutHorizontalTop = kDirHorizontal| kAlignTop,
207  kLayoutHorizontalCenter = kDirHorizontal| kAlignCenter,
208  kLayoutHorizontalBottom = kDirHorizontal| kAlignBottom,
209  kLayoutVerticalLeft = kDirVertical | kAlignLeft,
210  kLayoutVerticalCenter = kDirVertical | kAlignCenter,
211  kLayoutVerticalRight = kDirVertical | kAlignRight,
212  kLayoutOverlay = kDirOverlay | kAlignCenter,
213 };
